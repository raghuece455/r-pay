# Image Prompts for the R-Pay Article Series

Use these prompts for generated hero images or article illustrations. Do not use real payment product screenshots, logos, colors, or layouts from PhonePe, Google Pay, Paytm, BHIM, NPCI, banks, PagerDuty, Datadog, Grafana, Sentry, or other real products.

## Series Hero

Create a cinematic technical illustration for a Medium article series called "Claude Agents in Action: Building and Rescuing R-Pay". Show a fictional fintech sandbox with a mobile payment app on one side and a dark incident command dashboard on the other. Include architecture diagrams, metrics, logs, and human approval checkpoints. The style should be premium fintech and serious engineering. No real product logos, no real bank names, no NPCI marks, no copied UI.

## Part 1 Hero

Create a realistic engineering workspace illustration of a developer building a fictional Indian payment sandbox called R-Pay. Show a laptop with a Next.js app, payment state cards, API diagrams, and a UPI Switch Simulator block. The mood is calm build-week focus. Use original deep indigo, emerald, and warm off-white accents. No real payment brand logos or copied UPI app screens.

## Part 2 Hero

Create a dark-mode incident readiness illustration for R-Pay IncidentDesk. Show metrics, logs, traces, deployment history, runbooks, and AI incident analysis cards on large monitors. Human engineers are preparing runbooks before an incident. Serious SRE command center tone. No real incident tooling branding.

## Part 3 Hero

Create a midnight incident response scene for R-Pay. Show degraded payment health metrics, queue depth spike, DB pool saturation, and a suspicious worker deployment on a dark dashboard. Human engineers approve rollback while AI analysis cards summarize evidence. Cinematic, realistic, no real payment or observability product branding.

## Supporting Image: Payment State Machine

Create a clean visual diagram of a payment state machine for a fictional payment sandbox. Include CREATED, PROCESSING, SUCCESS, FAILED, PENDING, TIMED_OUT, and RECONCILED as cards connected by arrows. Make the SUCCESS transition visually guarded by "payment network confirmation". Minimal, editorial, technical.

## Supporting Image: Retry Storm

Create a technical illustration of a retry storm loop. Show a slow payment network simulator, a status reconciler polling every second, exploding queue depth, saturated database pool, and slow payment API. Use arrows and warning colors. Keep it serious and readable.
